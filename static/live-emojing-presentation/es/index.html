¿Por qué emojis?

  • Muy familiares
  • Antinaturales en el código
  • No tiene idioma
Encontré en los emojis la clave para atraer al público a leer código. Su atractivo surge de ser un elemento muy familiar y al mismo tiempo muy antinatural en los lenguajes de programación. Este contraste lo hace destacar.    Los emojis comunican rápida y fácilmente, y no hacen distinción de idioma.

Intereses

  • Participación de la audiencia
  • Todos podemos codear
  • Codear puede ser una actividad social
Además de mi interés en que el público participe en la presentación, tengo la intención    de derribar la idea de que la programación es inaccesible. Socializar el código invitando al    espectador para intervenir.

¿Cómo facilitar la lectura de código?

  • Tamaño de fuente
  • Página en blanco
  • Castellanización del código​​
My interest to make the audience take part in the livecoding experience brought to me the idea of “​ livecoding with emojis​​ “ I had previously tried to reach out for such interaction by other means.

Castellanización del código​​​​

I created aliases in our native language of some function names or code blocks to replace their original English names as a way to engage Non-English speakers during presentations. Displaying the code written in native language draws attention, eliminates the difficulty of the language barrier and focuses attention on what the code is really about. 
  					
  let pista1 = d1
      pista2 = d2
      sonido = sound
      cada_tanto = sometimes
      despacito = slow 4
      primeros_8 = run 8
      empezar_desde = begin
      comprimir = shape
  		cortar =  cut "1"

Prueba de Concepto

  				
  	emoji x = sound (fromString $ concatMap (\c) ->
      if c == '💚' then s "bd:6"
      else if c == '✊' then s "sn:4"
      else if c == '😬' then s "dr"
      else if c == '😎' then s "dr:1"
      else if c == '👄' then s "feel:1"
      else if c == '💋' then s "kiss:2"
      else if c == '😘' then s "kiss"
      else if c == '😸' then s "cat"
  	)

  	d1 $ emoji "💚(3,8) ✊*2"
